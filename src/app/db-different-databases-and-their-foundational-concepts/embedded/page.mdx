# Embedded Databases

## Table of Contents

## Introduction

Embedded databases are lightweight database systems that are integrated into applications, providing data storage and retrieval capabilities without the need for a separate database server. They are designed to be fast, efficient, and easy to use, making them ideal for applications that require local data storage, such as mobile apps, IoT devices, and desktop applications.

## Key Features of Embedded Databases

### Lightweight and Fast

Embedded databases are optimized for performance and resource usage, allowing them to run efficiently on devices with limited resources. They typically have a small footprint and low memory overhead, making them suitable for embedded systems.

Embedded databases are designed to be fast and efficient, often using in-memory storage or optimized disk access patterns. This allows for quick data retrieval and manipulation, which is crucial for applications that require real-time performance.

### Easy Integration

Embedded databases can be easily integrated into applications without the need for complex setup or configuration. They often provide simple APIs or libraries that developers can use to interact with the database directly from their code.
This ease of integration allows developers to focus on building their applications rather than managing a separate database server.

### Local Data Storage

Embedded databases store data locally within the application, eliminating the need for network communication to access data. This is particularly beneficial for applications that require offline access or need to operate in environments with unreliable network connectivity. Local data storage also improves performance by reducing latency associated with remote database access.

### ACID Compliance

Many embedded databases support ACID (Atomicity, Consistency, Isolation, Durability) properties, ensuring that transactions are processed reliably and data integrity is maintained. This is crucial for applications that require robust data management and consistency, such as financial applications or critical systems.

### Cross-Platform Support

Embedded databases are often designed to be cross-platform, allowing them to run on various operating systems and hardware architectures. This flexibility enables developers to build applications that can run on multiple devices without worrying about database compatibility issues.

## Data Models and Query Languages

Embedded databases can support various data models, including key-value stores, document stores, and relational models. The choice of data model depends on the specific requirements of the application and the type of data being stored.
Many embedded databases provide their own query languages or APIs for data manipulation. These query languages are often designed to be simple and intuitive, allowing developers to perform CRUD (Create, Read, Update, Delete) operations easily. Some embedded databases also support SQL-like query languages, enabling developers to leverage their existing SQL knowledge.

## Benefits of Using Embedded Databases

- **Performance**: Embedded databases are optimized for speed and efficiency, providing fast data access and manipulation capabilities. This is particularly important for applications that require real-time performance or need to handle large volumes of data.
- **Simplicity**: The ease of integration and setup makes embedded databases a convenient choice for developers. They can quickly add data storage capabilities to their applications without the need for complex configuration or management.
- **Local Data Access**: Embedded databases allow applications to access data locally, reducing latency and improving performance. This is especially beneficial for applications that require offline access or need to operate in environments with unreliable network connectivity.
- **Resource Efficiency**: Embedded databases are designed to be lightweight and resource-efficient, making them suitable for devices with limited resources. They typically have a small footprint and low memory overhead, allowing them to run efficiently on embedded systems.
- **Cross-Platform Compatibility**: Many embedded databases are designed to be cross-platform, enabling developers to build applications that can run on multiple devices and operating systems. This flexibility allows for greater portability and compatibility across different environments.

## Disadvantages of Embedded Databases

- **Limited Scalability**: Embedded databases may not be suitable for applications that require high scalability or handle large volumes of data. They are typically designed for local data storage and may not support distributed architectures or clustering.
- **Lack of Advanced Features**: Some embedded databases may lack advanced features found in full-fledged database management systems, such as advanced indexing, complex query optimization, or extensive data analytics capabilities.
- **Vendor Lock-In**: Depending on the embedded database chosen, there may be a risk of vendor lock-in, especially if the database uses proprietary APIs or formats. This can make it difficult to switch to a different database solution in the future.
- **Limited Community Support**: Some embedded databases may have smaller communities or less extensive documentation compared to larger database systems. This can make it more challenging to find support or resources when encountering issues or needing assistance with the database.

## How does an embedded database work?

An embedded database works by integrating the database engine directly into the application code, allowing the application to access and manipulate data without the need for a separate database server. The database engine is typically compiled into the application binary or linked as a library, enabling it to run in the same process as the application.

When the application needs to perform database operations, it uses the provided APIs or query languages to interact with the embedded database. The database engine handles data storage, retrieval, and management tasks, ensuring that data is stored efficiently and can be accessed quickly. Transactions, indexing, and other features are managed by the embedded database engine, providing a seamless experience for developers and users alike.

Embedded databases differ from traditional client-server databases in that they are integrated directly into the application rather than running as a separate server process. This integration allows for faster data access and reduced latency, as there is no need for network communication to access the database. Additionally, embedded databases are typically designed to be lightweight and resource-efficient, making them suitable for applications with limited resources or those that require local data storage.

## Common Use Cases

Embedded databases are commonly used in various scenarios, including:

- **Mobile Applications**: Many mobile apps use embedded databases to store user data, preferences, and application state locally. This allows for quick access to data without relying on network connectivity.
- **IoT Devices**: Embedded databases are often used in Internet of Things (IoT) devices to store sensor data, configuration settings, and device state. The local storage capabilities of embedded databases enable IoT devices to operate independently and efficiently, even in remote or disconnected environments.
- **Desktop Applications**: Desktop applications frequently use embedded databases to manage application data, such as user profiles, settings, and application state. This allows for quick data access and manipulation without the need for a separate database server.
- **Gaming Applications**: Many games use embedded databases to store game state, player profiles, and other game-related data. The fast performance and local storage capabilities of embedded databases make them ideal for gaming applications that require real-time data access and manipulation.
- **Embedded Systems**: Embedded databases are commonly used in embedded systems, such as automotive applications, medical devices, and industrial control systems. These systems often require reliable and efficient data storage solutions that can operate in resource-constrained environments.

## Popular Embedded Databases

### SQLite

SQLite is a widely adopted embedded database known for being self-contained, serverless, and requiring zero configuration. It stores all data in a single file, making it easy to manage and distribute across different platforms. SQLite supports a broad subset of SQL features and is frequently used in mobile apps, desktop applications, and embedded systems. SQLite offers strong SQL support, covering most of the standard language while introducing some unique features and omitting a few advanced ones. It provides flexible upsert operations through the `INSERT ON CONFLICT` clause, supports subqueries in various statements, and allows the creation of user-defined functions and aggregates. Its comprehensive set of built-in functions enables developers to perform a wide range of data transformations and computations directly within SQL queries.

```sql title="Example of SQLite Query"
SELECT
    name,
    age,
    city
FROM
    users
WHERE
    age > 30
ORDER BY
    age DESC;
```

This query retrieves the names, ages, and cities of users older than 30, ordering the results by age in descending order. SQLite's SQL capabilities allow for complex queries and data manipulation, making it a versatile choice for embedded applications.

SQLite is particularly well-suited for applications that require lightweight data storage and retrieval capabilities. It is often used in mobile applications, desktop software, and embedded systems due to its simplicity and efficiency. SQLite's self-contained nature allows developers to easily integrate it into their applications without the need for a separate database server. Its support for transactions, indexing, and complex queries makes it a powerful tool for managing application data.
SQLite is a popular choice for developers looking for a lightweight, easy-to-use embedded database solution.

### LevelDB

LevelDB is a fast key-value storage library developed by Google. It is designed for high performance and low memory usage, making it suitable for applications that require efficient data storage and retrieval. LevelDB provides a simple API for reading and writing key-value pairs and supports features like compression and batch writes. LevelDB is a high-performance key-value store that is optimized for fast read and write operations. It uses a log-structured merge-tree (LSM tree) data structure to efficiently handle large volumes of data. LevelDB is often used in applications that require low-latency access to data, such as web browsers, mobile apps, and embedded systems.

```sql title="Example of LevelDB Query"
const level = require('level');
const db = level('./mydb');
db.get('user:123', function (err, value) {
    if (err) {
        if (err.notFound) {
            console.log('User not found');
        } else {
            console.error('Error retrieving user:', err);
        }
    } else {
        console.log('User data:', value);
    }
});
```

This example demonstrates how to retrieve a value from LevelDB using its API. LevelDB does not use SQL for querying; instead, it provides a simple key-value interface for data access. LevelDB is particularly well-suited for applications that require fast, low-latency access to data. It is often used in scenarios where high write throughput and efficient storage are critical, such as in web browsers, mobile applications, and embedded systems. LevelDB's simplicity and performance make it a popular choice for developers looking for a lightweight key-value store.

### DuckDB

DuckDB is an in-process SQL OLAP database management system that is designed for analytical workloads. It provides a lightweight and efficient way to perform complex queries on large datasets. DuckDB supports SQL queries, indexing, and vectorized execution, making it suitable for data analysis tasks. DuckDB is particularly well-suited for analytical workloads and data science applications. It can handle large datasets efficiently and provides features like columnar storage and vectorized execution for improved query performance. DuckDB can be embedded in applications, allowing developers to perform complex data analysis without the need for a separate database server.

```sql title="Example of DuckDB Query"
SELECT
    country,
    AVG(salary) AS average_salary,
    COUNT(*) AS employee_count
FROM
    employees
WHERE
    department = 'Engineering'
GROUP BY
    country
ORDER BY
    average_salary DESC;
```

This query retrieves the average salary and employee count for the Engineering department, grouped by country, and orders the results by average salary in descending order. DuckDB's SQL capabilities allow for complex aggregations and grouping operations, making it a powerful tool for data analysis.

### LMDB (Lightning Memory-Mapped Database)

LMDB is a fast memory-mapped key-value database that provides high performance and low latency. It is designed for applications that require efficient data access and manipulation, such as web servers, messaging systems, and embedded systems. LMDB supports ACID transactions, multi-version concurrency control, and provides a simple API for data access.
LMDB is known for its simplicity and efficiency, making it a popular choice for applications that require fast data access without the overhead of a full-fledged database server. It uses memory-mapped files to provide fast access to data, allowing applications to read and write data directly from memory. This approach minimizes disk I/O and improves overall performance, especially for read-heavy workloads.

```sql title="Example of LMDB Query"

LMDB's memory-mapped approach and support for ACID transactions make it a powerful tool for managing application data.
